Contenido y utilidad de los archivos

Index.html

la página principal. Contiene noticias sobre la página, un encabezado con los enlances más importantes y un pie de pagina. En la parte derecha de la página hay enlaces para crear noticias y administrar estas que no se han habilitado.

Perfil.html

Aqui se reflejará toda la información del usuario y las listas guardadas. Se podran editar los datos personales y se verán las listas del usuario. Estas listas se pueden borrar indicando el número de la lista. También se pueden imprimir las listas click-ando en el botón de imprimir. La impresión solo imprime los productos de tu lista.

Productos.html

Aquí se mostrará la lista de los productos almacenados en el sistema mediante pestañas. También se da la posibilidad de guardar productos mediante un formulario que se expande.

Mis listas.html

Te da opción a crearte una lista de productos. Se van insertando los productos uno a uno y la lista va incrementando. Después podemos guardar la lista y esta aparecerá en nuestro perfil, siempre y cuando estemos registrados. Esta lista se puede expandir o contraer y podemos encontrar un producto en él.

Registro.html

La página para que nuevos usuarios puedan registrarse en el sistema. Contiene java script para validación.

Sesión.html

Página para poder identificarse en la página

Ubicación.html

Página donde se podrá encontrar el supermercado o supermercados más cercanos a tu ubicación. Contiene geolocalización. Podemos elegir un supermercado y nos muestra la ubicación de los supermercados en el mapa.

MyCss.css

Es el archivo encargado de darle estilo a todas las páginas

mootree.css

Css que utiliza el Jtree

map.css

Css para google map

jquery-ui-1.8.11.custom.css

Css para los botones, checkbox... que existen, con otros estilos.

Java Script

Existe básicamente un archivo JS para cada html. Después, el Jtree tiene su propio JS y 3 JS para Jquery y los diferentes themes

Ajax

Se ha usado Ajax para poder pasar los datos obtenidos mediante javascript en el Jtree al archivo listajax.php para su insercion

en la base de datos.

Base de datos

La base de datos usa 3 tablas: usuario, lista y productos, que están conectadas entre si mediante claves ajenas.

Conclusiones de la tercera entrega.

Por falta de tiempo no se han podido realizar varias ideas que desde un principio se tenían en mente. Toda la parte del administrador está sin implementar pero lo realizaría añadiendo un campo "isAdmin" a la tabla de usuarios y agregando un usuario admin a mano en la base da datos. Mediante la variable $_SESSION, cuando se iniciase la sesión como administrador, se verían enlaces que como usuario normal no se podrían ver, y en estos enlaces hacer las acciones de delete, update e insert contra la base de datos. Todo esto se realizaría para los usuarios, productos e listas.

Tampoco se ha implementado mediante googlemaps el camino más corto desde la posición del usuario al supermercado más cercano. No he encontrado información de como escoger el "mark" más cercano y tampoco como intregarlo con la geolocalización

Por último tampoco se ha implementado el resultado de la lista de la compra más barata. Lo realizaría haciendo varios SELECT contra la base de datos, seleccionando el producto con su unidad y precio donde los parametros serían los distintos supermercados. Despúes, tendría una variable para cada supermercado dónde se iría sumando el precio. AL final tendría el valor de la lista en dinstintos supermercados y con esto compararía y se lo mostraría al usuarios.

En mi opinión está asignatura ha sido de las más valiosas de la carrera porque he aprendido muchisimo sobre php, javascript Ajax y el uso de estas tecnologías para crear páginas web. Me parece una vergüenza que los lenguajes de programación web se tengan que dar en un seminario. Un Ingeniero informático debería de salir de la carrera sabiendolo.
